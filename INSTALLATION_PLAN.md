# План реализации Bash-скрипта для установки VLESS Self-Healing Gateway

Данный документ описывает архитектуру и логику работы скрипта `install.sh`, разработанного для автоматизации развертывания приложения на Ubuntu/Debian.

## 🎯 Цель
Обеспечить быструю и безошибочную установку всех необходимых компонентов: системных зависимостей, Xray-core, окружения Python и системных служб (systemd).

## 🛠 Технологический стек
- **Shell:** Bash
- **OS:** Ubuntu/Debian (основная поддержка)
- **Зависимости:** `python3`, `pip`, `openssl`, `curl`, `jq`, `Xray-core`
- **Управление службами:** `systemd`

## 📝 Логика работы скрипта (Workflow)

### 1. Валидация (Pre-flight checks)
- **Root Check:** Проверка, запущен ли скрипт с правами `sudo`.
- **OS Check:** Проверка наличия `/etc/debian_version` для гарантии совместимости.
- **Dependency Check:** Проверка наличия базовых утилит (`curl`, `openssl` и т.д.).

### 2. Подготовка системы
- **System Update:** Выполнение `apt update`.
- **Package Installation:** Установка `python3`, `python3-pip`, `openssl`, `jq`, `curl`.
- **Xray-core Installation:** Автоматический запуск официального скрипта установки Xray.

### 3. Конфигурация приложения
- **Environment Setup:** 
    - Копирование `.env.example` в `.env`.
    - **Security:** Автоматическая генерация 16-байтного ключа через `openssl rand -base64 16`.
    - **Injection:** Использование `sed` для безопасной вставки сгенерированного пароля в `.env` (с поддержкой спецсимволов через разделитель `|`).
- **Python Environment:** Подготовка к запуску Python-скриптов.

### 4. Настройка Systemd (Автоматизация Service Management)
- **Dynamic Path Injection:** 
    - Скрипт определяет текущий `pwd` (абсолютный путь к проекту).
    - Проходит по файлам `vless-gateway.service` и `vless-checker.service`, заменяя заглушку `/home/user/vless-checker` на реальный путь к проекту. Это критически важно для работы `WorkingDirectory`.
- **Deployment:**
    - Копирование обработанных файлов в `/etc/systemd/system/`.
    - Копирование таймера `vless-checker.timer`.
- **Activation:**
    - `systemctl daemon-reload`.
    - `systemctl enable --now` для сервиса шлюза и таймера проверки.

### 5. Верификация и вывод
- **Status Check:** Использование `systemctl is-active` для подтверждения запуска основных компонентов.
- **User Feedback:** 
    - Вывод итогового пути проекта.
    - Инструкции по просмотру логов через `journalctl`.
    - Предупреждение о том, где искать сгенерированный пароль.

## ⚠️ Обработка ошибок
- Использование `set -e` для немедленной остановки при любой ошибке.
- Кастомные функции `log()`, `warn()`, `error()` для цветного и понятного вывода в терминал.
- Информативные сообщения при падении сервисов (подсказки по командам отладки).
